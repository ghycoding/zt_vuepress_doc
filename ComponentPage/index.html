    # 全局组件

    # 一、select-customize-new

    组件说明：此组件基于element-ui 中的el-select 进行了封装，主要功能有可以实现前端搜索，分页加载，自定义显示内容等；主要根据业务需求开发；
    和【select-customize】组件不同的是新版新增了可以自定义绑定key，可以自定义输出的字段，新的props参数如下

        // 组件默认接收的参数
    selectOptionSet: {
      type: Object,
      default: function () {
        return {
          key: 'id',  // 唯一标识
          label: "title",  // 显示的label
          outputField: 'id', // 输出的字段名称  就是在双向绑定的时候接收的值可以进行指定字段；比如在这里指定的是id就接收的id  ，指定title就接收title ;接收的值是一个一维数组
          checkAllNumber: 2000, // 全选条数，默认2000，如果为0表示不限制
        }
      }
    }

    在页面中调用，需引入注册，需要注意的是在这里绑定需要使用 :value.sync， 传入的value是一个对象

       <SelectCustomizeNew multipleFlag 
    :data="channel_id" @change="getChannelId"
      :value.sync="searchForm.channel"
      :selectOptionSet="{ key: 'id', label: 'title', outputField: 'id',checkAllNumber:0}"
      ref="channelId"/>

    * 必须要有appInfo和appInfoKey这两个字段 ， appInfo 在组件中存储输出的值；appInfoKey主要来存储输入的值，作为回显，如果有默认值的话需要把默认值push到appInfoKey字段中来进行回显；如果把默认值赋值到appInfo中可能回显不会生效；组件接收value的值如下：

    data(){
  return{
    searchForm:{
      channel:{
        appInfo:[],  
        appInfoKey:[]
      }
    }
  }
}
